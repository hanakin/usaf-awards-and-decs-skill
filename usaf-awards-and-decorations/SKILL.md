# USAF Awards and Decorations

## Overview

Turn raw accomplishments into clear, defensible award and decorations content. Pull out mission impact, leadership value, and measurable results while keeping the final draft concise and easy for a supervisor to review.

## Workflow

1. Collect the basic facts before drafting. Use [references/intake-checklist.md](references/intake-checklist.md) to gather rank, duty title, organization, award type, period of performance, major accomplishments, metrics, and any local formatting rules.
2. Identify the target output before writing. Use [references/output-formats.md](references/output-formats.md) to choose the right structure for a medal citation, 1206-style package, quarterly or annual nomination, or a short leadership summary.
3. Rewrite accomplishments into action, impact, result form. Use [references/writing-patterns.md](references/writing-patterns.md) to tighten verbs, add scale, and eliminate vague praise.
4. Build the draft from the strongest evidence first. Prioritize mission effect, readiness, money saved, time saved, inspection results, Airmen developed, and commander-level outcomes.
5. Flag uncertainty instead of inventing facts. If metrics, dates, or award-specific rules are missing, keep placeholders and call out exactly what still needs confirmation.

## Drafting Rules

- Lead with achievements, not biography.
- Prefer quantified impact over adjectives.
- Keep claims proportional and believable.
- Preserve official names, units, dates, and capitalization exactly as provided.
- Avoid unsupported superlatives unless the user supplies evidence.
- Treat base, MAJCOM, wing, and local templates as controlling when they conflict with this skill.

## Working Style

- If the user provides rough notes, first normalize them into a fact list grouped by accomplishment.
- If the user provides a finished draft, edit for sharper impact, cleaner syntax, and better award-board readability without changing the underlying facts.
- If the user does not name the award format, infer the most likely one from the request and say what assumption was made.
- If the task looks compliance-sensitive or template-sensitive, ask for the local template only when the missing format would materially change the output.
