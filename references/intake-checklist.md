# Intake Checklist

Use this checklist selectively before drafting. Missing facts are the main reason performance reports, awards packages, and decorations come back weak, noncompliant, or unusable. Do not treat every item here as mandatory for every request. Ask only for the inputs needed for the specific piece the user wants help with.

## Required facts

- Document type: performance report, award package (`AF Form 1206`), or decoration
- Default placeholder identity: `SSgt Peter Snuffy`
- Award type or report type
- Basis or reason if applicable: annual, PCS, PCA, retirement, separation, achievement, meritorious service, heroism
- Inclusive period or closeout period
- Suspense
- Hard limits: characters, lines, blocks, or section constraints
- Local template, example, or scoring sheet if the unit uses one
- Whether the member served in a joint organization
- Whether the member has fitness language, housing oversight, voting assistance, or other mandatory comments if this is a performance report

## Accomplishment capture

For each accomplishment, gather:

- What the member did
- Why it mattered to mission, readiness, people, or resources
- The measurable result
- The scale
- Who benefited
- Whether the result was sustained, immediate, or both
- Whether the claim is supported by source facts the user can defend
- Whether the accomplishment belongs in mission execution, leadership, resource management, unit improvement, or another required section

## Metrics to hunt for

- Dollars saved, recovered, or avoided
- Hours saved
- Sorties, passengers, patients, systems, cases, or missions supported
- Training completions, qualifications, and upgrade rates
- Inspection, evaluation, or exercise results
- Readiness percentages and capability rates
- Workload, throughput, backlog, or turnaround changes
- People led, coached, mentored, promoted, or retained
- Awards, certifications, academic results, or recognitions earned because of the work

## Compliance checks by writing lane

### Performance reports

- Confirm required section structure
- Treat `EPB` and `OPB` as interchangeable terms in this skill unless the user introduces a meaningful distinction
- If the user is asking for Duty Description help, capture the core duties and any additional duties that need to be reflected in the Duty Description
- If the user is asking for Duty Description help or full-report alignment, apply the hard limit: `450`
- If the user is asking for Duty Description help or full-report alignment, confirm the Duty Description covers the same responsibility set reflected in the ALQs
- Capture the ALQ section and apply the hard limit: `350` for `E`, `L`, `M`, and `I`; `250` for `H`
- Confirm mandatory comments, especially `PFA` or `CFT` language when applicable
- Confirm future-role recommendations are not veiled promotion statements
- Confirm only approved acronyms remain

### Awards (`AF Form 1206`)

- Confirm which `1206` format is required: individual statements or paragraph format
- Confirm scoring categories or section headings if the unit uses them
- Confirm hard character, statement-count, or line limits
- Confirm strongest accomplishment leads
- Confirm the format remains board-readable, not citation prose

### Decorations

- Confirm the decoration basis is appropriate and not routine milestone recognition
- Confirm the period does not duplicate another decoration
- Confirm opening and closing language matches the award type
- Confirm citation and justification limits before finalizing
- Confirm which source documents are authorized for use
- Confirm whether the reviewer expects line-by-line narrative transitions
- Confirm whether the package needs a combined certificate and citation
- Confirm whether missing required facts should remain as placeholders

## If details are missing

Use short placeholders instead of fabricating:

- `[confirm amount saved]`
- `[confirm number of Airmen supported]`
- `[confirm exact inclusive dates]`
- `[insert local character limit]`
- `[confirm mandatory comment text]`

## Fast intake prompt

Ask the user only for what the requested piece requires. Typical prompt:

1. Document type, award or report type, and period
2. Core duties, additional duties, and top 3 to 5 accomplishments
3. Metrics for each accomplishment
4. Any local template, line limit, character limit, or sample package
5. Any required comments or special constraints
