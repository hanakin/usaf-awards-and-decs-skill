# Writing Patterns

## Core formula

Default to this sequence:

`Action -> Mission impact -> Result`

Example shape:

- Directed X, enabling Y, resulting in Z
- Overhauled X process, improving Y, which produced Z

This is a drafting pattern, not a license to force every output into the same sentence shape. Match the final wording to the actual writing lane: performance report, `AF Form 1206`, or decoration.

## What strong award language does

- Uses concrete verbs
- Connects the accomplishment to a mission outcome
- Shows scale or consequence
- Stays compressed enough for board review
- Uses plain language unless an approved acronym clearly improves readability
- Stays supportable and proportional to the facts

## Verb bank

Use verbs like:

- led
- directed
- rebuilt
- accelerated
- synchronized
- restored
- executed
- modernized
- resolved
- developed
- trained
- secured
- validated
- optimized

Avoid weak openings like:

- helped with
- was responsible for
- assisted in
- worked on

Avoid inflated language like:

- unparalleled
- unmatched
- best ever
- world class
- game changing

unless the user provides evidence that supports the claim and the format tolerates it, or the approved example set establishes that the phrasing is acceptable for that lane.

## Upgrade vague claims

Instead of:

- improved morale

Prefer:

- launched recognition program for 42 Airmen, raising volunteer participation by 30 percent

Instead of:

- did an outstanding job during inspection

Prefer:

- orchestrated inspection response across 5 sections, closing 18 discrepancies and helping secure a zero-major-finding result

## Priority order for content

When space is limited, prefer:

1. Mission execution and readiness
2. Quantified impact on operations, money, or time
3. Leadership and development of people
4. Process improvements
5. Community involvement or extra duties, if the award type expects them

## Lane-specific application

### Performance reports

- Keep statements in the section they belong to
- Emphasize sustained performance over the reporting period
- Preserve plain-English readability
- Keep mandatory comments separate from advocacy language
- Use ALQ-aware phrasing when the user is building performance-report statement options
- When asked for multiple options, vary structure, phrasing, and verb choice across the set
- Keep the member name near the front when the statement style expects it
- Treat the approved example library as an authoritative pattern source for acceptable EPB voice
- `H` is intentionally different: write it as a push statement instead of forcing the same action-impact-result balance used in `E`, `L`, `M`, and `I`
- For `H`, stronger recommendation language and future-readiness language are acceptable when they follow the approved example pattern
- Use `statement-shared.md` for common mechanics and `epb-guidance.md` for performance-report limits

## Performance-report statement workflow

- Start with a fuller summary of the accomplishment before compressing it
- Reduce the summary into one or more standalone statements
- Preserve exact metrics where available
- Validate acronym use, character limits, and clarity before presenting options
- Optimize to `350` characters for `E`, `L`, `M`, and `I`, and `250` for `H`
- For `H`, optimize for concise push and readiness language rather than mirroring the structure of the other ALQs

### Awards (`AF Form 1206`)

- Lead with the strongest board-relevant accomplishment
- Compress aggressively without making the statement cryptic
- Keep the action, impact, and result chain obvious
- For individual-statement `1206`, optimize each statement to `195-220`
- For individual-statement `1206`, match the statement count required by the specific award or template
- For paragraph-format `1206`, optimize to the required line count without forcing statement-style compression
- Use `1206-guidance.md` for both formats and their constraints

### Decorations

- Use narrative prose, not bullets
- Match the award-specific opening and closing sentence pattern
- Keep the tone formal and dignified without sounding automatic
- Avoid repeating the exact language of another decoration for the same period
- Expand source bullets into fact-based narrative rather than copying them verbatim
- Preserve the decisive metrics and mission effects from the selected accomplishments instead of summarizing them into vague praise
- Use transition phrases only when they improve flow or when the required format expects them
- Spell out organizations, operations, and terms instead of shortening them
- Spell out ranks in decoration prose rather than leaving abbreviated forms
- Avoid abbreviation-heavy writing even when the underlying source notes use shorthand

## Citation-specific reminders

- Citation text must follow award-specific structure from `DAFMAN 36-2806`
- `MSM`, `AFAM`, and `AFCM` citations are character-limited in this skill's current medal set
- `BSM` and higher citations are line-limited
- Use exact counting and acronym validation before finalizing
- Run spelling and grammar review before finalizing
- Reduce passive voice where it weakens force or clarity
- Treat abbreviation removal as part of final citation cleanup

## Editing checklist

- Remove filler adjectives
- Replace passive voice with direct action
- Add missing scale
- Shorten long setup clauses
- Keep one main idea per line unless the format requires denser wording
- Confirm every claim traces back to user-provided facts
- Confirm the wording matches the policy-governed format
- Validate hard length limits before delivery
- Validate acronym use before delivery
