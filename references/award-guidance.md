# Award Guidance

Use this reference for `AF Form 1206` work after reading `shared-guidance.md`.

This lane is assistive by default. If the user asks for one statement, provide one statement. If the user asks for a paragraph-format package, provide the requested paragraph block or blocks.

## Supported 1206 formats

`AF Form 1206` work in this skill supports two formats:

1. Individual statement format
2. Paragraph format with line-count limits that vary by award or template

Default to individual statement format unless the user or template explicitly requires paragraph format.
Always identify which format is required before drafting.

## Individual statement format

- Each statement must stay between `195-220` characters
- The `- ` prefix counts toward the total
- Count the complete string, including letters, numbers, whitespace, punctuation, symbols, and special characters
- Match the required number of statements for the specific award or template

## Paragraph format

- Use narrative paragraph blocks instead of standalone statements
- Follow the line-count limit required by the specific award or template
- Do not force the `195-220` character rule onto paragraph-format `1206` packages
- Preserve strong accomplishment ordering and board readability even when working in paragraph form
- Match the required total line count for the specific award or template

## Award-specific behavior

- Lead with the strongest mission-relevant accomplishment
- Keep the action, impact, and result chain obvious
- Preserve as much measurable detail as the limit allows
- Treat approved `1206` examples as authoritative style references for this skill
- Use the examples for structure, tone, ranking logic, and statement shaping while still validating against the active format's required length and count rules
- For paragraph format, compress by line economy instead of statement-character economy

## Review workflow from the source guidance

1. Identify whether the request is statement format or paragraph format
2. For statement format, build one final statement from the provided achievements unless the user explicitly asks for multiple options
3. For paragraph format, build the full paragraph block or blocks requested by the format
4. Validate length, acronym use, name format, repetition, spelling, grammar, clarity, and impact
5. Revise until the user approves

At each step, use the approved `1206` examples to show what strong statement shaping, ranking logic, and board-readable compression look like.

Use `shared-validation.md` for the full validation order, troubleshooting logic, impactfulness checklist, clarity checklist, and conflict-resolution priority.

## Output notes

- For statement format, present a single statement by default unless the user asks for multiple options
- Match the exact final suffix format only if the user or template requires it
- Do not assume quality-ranking suffixes are always required just because they appeared in the source examples
- For paragraph-format `1206`, present the draft in paragraph blocks and validate against line-count limits rather than statement-character limits
