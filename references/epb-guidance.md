# Performance Report Guidance

Use this reference for performance-report statement work after reading `statement-shared.md`. In this skill, `EPB` and `OPB` are treated as interchangeable terms unless the user later requires a distinction.

## Performance-report rules

- Use ALQ-based routing: `E`, `L`, `M`, `I`, and `H`
- Keep the member name near the front when the format expects named statements
- Keep statements in the correct section instead of blending multiple ALQs without a reason
- Preserve plain-English readability over compressed board slang
- Treat approved example language as valid guidance for acceptable EPB style in this skill
- Treat `H` as a push statement that can be more recommendation-oriented than `E`, `L`, `M`, and `I`
- For `H`, it is acceptable to emphasize readiness for greater responsibility when that matches the requested style and approved examples

## Hard length rules

- Duty Description: `450` characters
- `E`, `L`, `M`, and `I`: `350` characters
- `H`: `250` characters

Count the entire string, including letters, numbers, whitespace, punctuation, symbols, and special characters.

## Duty Description rules

- Treat Duty Description as a required performance-report paragraph block
- Keep Duty Description to `450` characters
- Summarize the member's key responsibilities, not just one standout event
- Ensure the responsibilities reflected in the ALQs are also represented in the Duty Description
- Include additional duties when they materially affect the member's evaluated role or appear in the ALQ content
- Keep the Duty Description broad enough to frame the report, but specific enough that the ALQ statements clearly fit under it
- Do not write the Duty Description like a medal citation or a `1206` statement
- Keep the Duty Description in plain, readable prose rather than overly compressed board shorthand

When the user asks for Duty Description help, focus on that block only. Do not assume they also want ALQ statements unless they ask for them.

## Duty Description examples

Use these to show what right looks like when building a Duty Description from a corresponding EPB accomplishment set.

### Example 1

`As Out of Access NCOIC, TSgt Snuffy inprocesses & oversees Airmen awaiting clearances, coordinates Sq events, & manages accountability and development of 25 flight members. He is also responsible for the Sq IQT planning team for 8 work roles & managing a training pipeline for out-of-access airmen in support of the Sq stan eval mission. 1N8 lead for 5-member exercise planning team that coordinates with HHQ to meet ACC & CCMD readiness requirements.`

Why this is right:

- captures the core responsibilities behind the corresponding `E`, `L`, and training-related content
- keeps the focus on duties and scope rather than single standout achievements
- includes the additional responsibility that materially belongs in the report

### Example 2

`As Joint Targeting Lead TSgt Snuffy fills an 0-3 OIC billet managing all target nominations and development for a 131-member task force, ensuring approval of all targets to USCYBERCOM and DoD requirements. He serves as the Air Force functional manager for targeting, advising GP leaders and mentoring 5 target analysts across 7 USCYBERCOM organizations. Additionally, as Deputy Flight Chief, he administers and mentors the largest flight in the SQ.`

Why this is right:

- frames the member's primary mission role first
- includes the broader functional-manager responsibility reflected in the ALQs
- adds the flight-chief responsibility because it materially affects the evaluated role

### Example 3

`As Chief of Joint Targets, TSgt Snuffy fills an O-3 role managing targeting development, SOPs, TTPs & provides inputs to intel & targeting policies, doctrine & requirements for a 250-member JTF. Additionally, he serves as the AF targeting functional manager advising AF & Joint leaders, while mentoring 4 Amn across 4 USCYBERCOM organizations. Lastly, as Deputy Flight Chief for 39-members, he provides trained & equipped combat ready Amn to 2 JTFs.`

Why this is right:

- covers the operational, doctrine, and policy responsibilities tied to the related ALQ statements
- keeps mission duties ahead of mentoring and flight leadership
- shows how broad responsibility, force development, and trained-and-equipped support can all coexist in one Duty Description

## Guardrails

- Do not force `1206`-style dash bullets into a performance report unless the provided format requires it
- Do not append quality-ranking suffixes unless the requested output explicitly uses them
- Keep mandatory comments and evaluation requirements from `AFI 36-2406` separate from statement advocacy
- Do not flatten `H` into the same tone as the other ALQs; preserve its push-statement role

## Workflow

1. Review the provided achievements and determine what performance-report help the user is asking for
2. If the user is asking for ALQ-category help, determine which ALQ category fits best
3. If the user is asking for Duty Description help, build or revise the Duty Description so it covers the responsibilities reflected across the ALQs or accomplishment set provided
4. If the user is asking for statement help, confirm the target section before drafting
5. Build one statement at a time for that section
6. Generate variations for that single statement when the user asks
7. Validate exact count, acronym use, spelling, grammar, clarity, and section fit before delivery
8. When both Duty Description and ALQ content are in scope, validate Duty Description alignment before delivery

At each step, use the approved performance-report examples to show what a good ALQ fit, good split, good push statement, or good final statement looks like.

Use `statement-validation.md` for the detailed validation order, troubleshooting logic, and conflict-resolution priority.

Use `performance-report-examples.md` to compare ALQ fit, one-statement-at-a-time patterns, and approved `H` push-statement style before drafting.
