# Performance Brief Intake

Use this file for performance-brief intake.

## Default intake

Ask only for the inputs needed for the requested performance-brief help.

If the user wants a full performance brief or alignment help, gather:

- the Duty Description or the duty facts needed to build it
- the ALQ statements or the achievements that need to be placed into `E`, `L`, `M`, `I`, or `H`
- any required comments, especially fitness language when applicable
- any local template or block constraints

## Duty Description intake

When the user wants Duty Description help, gather:

- primary duties
- additional duties that materially belong in the Duty Description
- any scope counts such as people, programs, work roles, or mission sets

## Statement intake

When the user wants one or more ALQ statements, gather:

- the target ALQ if known
- the raw accomplishment facts
- the measurable effect
- the relevant scale
- any character limit that applies

## Hard limits to remember

- Duty Description: `450`
- `E`, `L`, `M`, and `I`: `350`
- `H`: `250`

## Default restrictions

Do not ask for unrelated full-package data when the user wants only:

- one Duty Description
- one ALQ statement
- ALQ-category help
- cleanup of existing performance-brief text
